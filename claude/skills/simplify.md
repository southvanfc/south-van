# Skill: /simplify

Reviews changed code for reuse, quality, and efficiency, then fixes any issues found.

## When to use

- After editing components to catch redundant markup or styles
- After adding CSS to `src/styles/global.css` to ensure new rules aren't duplicating existing utilities
- After building a new feature to check for unnecessary complexity

## Example triggers

- Just finished a new component → run `/simplify` to review it
- Added new styles and want to check for drift from the design system
